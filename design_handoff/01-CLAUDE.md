## Locked decisions

These four decisions were made by Marrs ahead of the build. Do not revisit them without asking.

### Stack: Next.js (App Router)

Use Next.js with the App Router. The JSX in `design_handoff/designs/` ports almost 1:1. TypeScript is recommended but not mandatory. No CSS framework unless it has a clear win; the prototype's inline styles and `tokens.css` port cleanly to CSS Modules or vanilla CSS.

### LLM: Minimax (via OpenRouter) for v1

Use Minimax for Phase C conversation and Blueprint generation. Marrs has a capped pricing plan that makes this the right cost profile for v1. Route all LLM calls through server-side Next.js API routes. Do not expose API keys to the browser.

Anthropic API is the upgrade path for future quality improvements. Build the LLM call layer behind a thin provider abstraction so the swap is a one-file change later. Note: Marrs's Claude Max subscription is for Claude Code usage during development, not for production API calls.

### Shape library: all 8 CWU shapes

Use the 8-shape library from `config/cwu-shapes.json`, backed by the methodology in `design_handoff/reference/CWU_Design_System_v0_1.pdf`.

The prototype's `SHAPE_LIBRARY` in `agents/phase-b.jsx` has 5 shapes (Pipeline, Studio, Ops-heavy, Platform, Solo-first). These map to the 8 CWU shapes per `design_handoff/06-shape-reconciliation.md`. During the build, expand the prototype's 5 to match the full 8. The prototype's heat map row patterns and timings stay the same, the shape set widens.

### Pricing bands in the Blueprint

Indicative baseline bands for the Blueprint's Map / Transform / Operate section:

- **Map:** from $5,000 AUD
- **Transform:** from $10,000 AUD (scales with team size)
- **Operate:** from $999 AUD per month (scales with team size)

These are v1 placeholders. Marrs will provide a refined pricing document with proper per-shape and per-team-size calculations before launch. Plumb pricing through a `lib/pricing.ts` module so the logic upgrade is isolated. Persist a `pricing_version` field on each Blueprint so historical Blueprints don't break when pricing logic changes.

## Non-negotiables

### 1. Brand tokens
All of `design_handoff/designs/shared/tokens.css` values are load-bearing. Coral is human, amber is hybrid, mint is agent. Don't remap.

### 2. Copy
Every piece of user-facing copy in the HTML is final. **Do not paraphrase.** If copy feels off to you, flag it; do not silently rewrite it.

### 3. No em-dashes in user-facing copy
Per original `design_handoff/designs/uploads/CLAUDE.md` §16.1. Check all strings you add, including anything generated by Minimax. Enforce in the LLM system prompts.

### 4. The flow is a narrative
`/agents` is three phases with specific timing:
- Phase B's dramatic reveal **must** include the one-cell-at-a-time animation and the 4-second pause before the chat nudge pulses in
- Phase C **must** preserve the system prompt shape that conditions the LLM on the user's answers + derived team

Don't shortcut.

### 5. Persistence across reloads
All forms of state (Phase A answers, Phase B derived data, Phase C messages) must survive a page reload during the flow. The prototype uses `localStorage`; production uses Supabase-backed sessions. Generate a session uuid on Phase A start, store in a cookie, all writes go through the server.

## Blueprint generation

In the prototype, `blueprints/blueprint-data.js` has a static mapping of role → description. In production:

1. At Phase C completion, trigger a Minimax call that generates the full Blueprint JSON (agent names, agent descriptions, day-in-the-life narrative, heat map narrative) conditioned on the user's answers + derived team + shape.
2. Persist the generated JSON to `blueprints.content` in Supabase.
3. `/blueprints/[id]` reads the JSON and renders. No regeneration on read.
4. Keep `AGENT_DETAIL` from `blueprint-data.js` as a static fallback for failed Minimax calls.

See `design_handoff/reference/CWU_Design_System_v0_1.pdf` Part 4 for the pre-mapper inference logic that drives Blueprint content generation.

## What success looks like

A user can:
1. Land on the homepage, read through, hit "map your business"
2. Complete 11 questions (5 min)
3. Watch the heat map reveal
4. Chat with their agent team
5. Be emailed a link to their blueprint
6. Open the blueprint, share with a teammate, book a call

Every step feels like the prototype. Every brand token is honored. Every piece of copy is exact. The Blueprint shows the visitor an 8-shape informed CWU designed for their specific business.
